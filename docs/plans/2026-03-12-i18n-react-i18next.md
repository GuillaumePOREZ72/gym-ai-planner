# i18n (react-i18next) Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add full EN/FR internationalisation to the frontend using `react-i18next`, with an EN/FR toggle button in the Navbar persisted to `localStorage`.

**Architecture:** Install `react-i18next`, `i18next`, and `i18next-browser-languagedetector`. Create `frontend/src/i18n/index.ts` (init config) and two JSON locale files (`en/common.json`, `fr/common.json`). Wrap every hardcoded UI string in the 10 in-scope files with `t("namespace.key")`. Add a language toggle button in Navbar modelled on the existing theme toggle.

**Tech Stack:** React 19, TypeScript strict, `react-i18next` ^15, `i18next` ^24, `i18next-browser-languagedetector` ^8, Vite (no SSR — browser-only detection is fine).

## Key conventions

- Single namespace: `"common"` (all keys in `common.json`)
- Key format: `"section.camelCaseKey"` — e.g. `"fitness.title"`, `"nav.dashboard"`
- Hook usage: `const { t } = useTranslation("common");` at top of each component
- Language stored in `localStorage` key `"i18n_lang"` (separate from `"theme"`)
- Fallback language: `"en"`
- Toggle button: text `EN | FR` in Navbar, same pattern as theme toggle (`<button>` with `aria-label`)
- No i18n in dynamic AI-generated content (plan text, report text, insights) — those come from the API as-is

## Task 2: Create i18n config (`frontend/src/i18n/index.ts`)

**Files:**
- Create: `frontend/src/i18n/index.ts`

**Step 1: Create the file**

```ts
// frontend/src/i18n/index.ts
import i18n from "i18next";
import { initReactI18next } from "react-i18next";
import LanguageDetector from "i18next-browser-languagedetector";

import en from "./locales/en/common.json";
import fr from "./locales/fr/common.json";

i18n
  .use(LanguageDetector)
  .use(initReactI18next)
  .init({
    resources: {
      en: { common: en },
      fr: { common: fr },
    },
    fallbackLng: "en",
    supportedLngs: ["en", "fr"],
    detection: {
      order: ["localStorage", "navigator"],
      lookupLocalStorage: "i18n_lang",
      caches: ["localStorage"],
    },
    interpolation: {
      escapeValue: false, // React already escapes
    },
  });

export default i18n;
```

**Step 2: Import it in `frontend/src/main.tsx`**

Add this import as the very first line (before `./index.css`):

```ts
import "./i18n/index.ts";
```

Result:
